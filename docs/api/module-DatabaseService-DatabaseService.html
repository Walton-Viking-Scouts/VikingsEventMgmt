Class: DatabaseService

DatabaseService~DatabaseService()

SQLite Database Service for offline data persistence

Manages all local data storage for the Vikings Event Management application. Automatically detects platform capabilities and falls back to localStorage when SQLite is not available. Provides consistent API across all platforms with support for Scout sections, events, attendance, and member data.

Constructor

new DatabaseService()

Creates a new DatabaseService instance

Initializes the service with platform detection and default state. Does not establish database connection - call initialize() for that.

(async) initialize() → {Promise.<void>}

Initializes the database service and creates necessary tables

Detects platform capabilities and establishes appropriate storage mechanism. On native platforms, initializes SQLite with proper connections and creates database schema. On web platforms, validates localStorage availability and sets up fallback mode. Safe to call multiple times - subsequent calls are ignored.
